# Email Setup Guide for Contact Form

The contact form is configured to send emails to **2 email addresses**. You have three options:

## Quick Comparison

| Option | Backend Required | Setup Difficulty | Best For |
|--------|-----------------|------------------|----------|
| **EmailJS** | No | Easy | Static sites, quick setup |
| **PHPMailer** | PHP server | Medium | PHP hosting, full control |
| **Node.js** | Node.js server | Medium | Node.js hosting, modern stack |

## Option 1: EmailJS (Recommended for Static Sites - No Backend Required)

### Setup Steps:

1. **Sign up at EmailJS**: https://www.emailjs.com/ (Free tier available)

2. **Create an Email Service**:
   - Go to Email Services → Add New Service
   - Choose your email provider (Gmail, Outlook, etc.)
   - Connect your email account

3. **Create an Email Template**:
   - Go to Email Templates → Create New Template
   - Use this template:
     ```
     Subject: New Contact Form - {{project_type}}
     
     From: {{from_name}} ({{from_email}})
     Phone: {{phone}}
     Company: {{company}}
     Project Type: {{project_type}}
     
     Message:
     {{message}}
     ```
   - Set "To Email" to: `{{to_email}}`

4. **Get Your Keys**:
   - Public Key: Dashboard → Account → API Keys
   - Service ID: Email Services → Your Service
   - Template ID: Email Templates → Your Template

5. **Update `assets/main.js`**:
   - Replace `YOUR_PUBLIC_KEY` with your EmailJS public key
   - Replace `YOUR_SERVICE_ID` with your service ID
   - Replace `YOUR_TEMPLATE_ID` with your template ID
   - Update `email1@aipulse.tech` and `email2@aipulse.tech` with your actual email addresses

6. **Configure Template for 2 Recipients**:
   - In EmailJS template, you can add both emails in the "To Email" field separated by comma
   - Or create 2 separate template calls (already implemented in the code)

## Option 2: PHPMailer (PHP Backend - Recommended for PHP Hosting)

### Setup Steps:

1. **Install PHPMailer**:
   ```bash
   cd server
   composer install
   ```
   If you don't have Composer, see `server/PHPMailer_SETUP.md` for manual installation.

2. **Configure SMTP**:
   - Edit `server/contact-handler.php`
   - Update the `$config` array (lines 25-35):
     ```php
     'smtp_host' => 'smtp.gmail.com',
     'smtp_port' => 587,
     'smtp_username' => 'your-email@gmail.com',
     'smtp_password' => 'your-app-password',
     'recipient_1' => 'email1@aipulse.tech',
     'recipient_2' => 'email2@aipulse.tech',
     ```

3. **Gmail App Password** (if using Gmail):
   - Enable 2-Step Verification
   - Generate App Password: Google Account → Security → App passwords
   - Use this password in `smtp_password`

4. **Upload Files**:
   - Upload `server/contact-handler.php` to your PHP server
   - Upload `server/vendor/` directory (generated by composer)

5. **Test the Form**:
   - The frontend is already configured to use PHP
   - Just update the path in `assets/main.js` if needed

See `server/PHPMailer_SETUP.md` for detailed instructions.

## Option 3: Node.js Backend SMTP Server

### Setup Steps:

1. **Install Dependencies**:
   ```bash
   cd server
   npm install
   ```

2. **Configure Environment**:
   - Create `server/.env` file
   - Add your SMTP credentials:
     ```
     SMTP_HOST=smtp.gmail.com
     SMTP_PORT=587
     SMTP_USER=your-email@gmail.com
     SMTP_PASSWORD=your-app-password
     EMAIL_1=email1@aipulse.tech
     EMAIL_2=email2@aipulse.tech
     PORT=3000
     ```

3. **Gmail App Password** (if using Gmail):
   - Enable 2-Step Verification
   - Generate App Password: Google Account → Security → App passwords
   - Use this password in `SMTP_PASSWORD`

4. **Start the Server**:
   ```bash
   npm start
   ```

5. **Update Frontend** (if using different URL):
   - In `assets/main.js`, update the fetch URL:
     ```javascript
     const response = await fetch("http://your-server-url/api/contact", {
     ```

6. **Deploy Backend**:
   - Deploy to Railway, Render, Vercel Functions, or your own server
   - Update the API URL in the frontend code

## Testing

1. Fill out the contact form
2. Submit the form
3. Check both email inboxes for the submission
4. Verify the auto-reply email is sent to the user

## Current Configuration

- **Form sends to**: 2 email addresses (configurable)
- **Auto-reply**: Enabled (sends confirmation to user)
- **Form fields**: Name, Email, Phone, Company, Project Type, Message

## Troubleshooting

- **EmailJS not working**: Check browser console for errors, verify API keys
- **Backend not working**: Check server logs, verify SMTP credentials
- **Emails not received**: Check spam folder, verify email addresses are correct
